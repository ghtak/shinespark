# Configuration Guide (`AppConfig`)

Shinespark 애플리케이션은 유연하고 강력한 계층적 설정 시스템을 지원합니다. 설정은 TOML 파일과 `.env` 환경 변수 파일을 통해 관리되며, 오버라이딩(덮어쓰기) 규칙에 의해 최종 값이 결정됩니다.

## 1. 파일 위치 및 로딩
* **설정 파일 (`.toml`)**: `configs/` 디렉토리 하위에 위치합니다.
* **환경 변수 파일 (`.env`)**: 런타임 환경에 따라 실행 파일 위치 혹은 프로젝트 루트(Workspace)에서 검색하여 로드합니다.

부트스트랩 시 `AppConfig::load_dotenv()`와 `AppConfig::new()`를 차례로 호출하여 두 가지 소스(File & Env)를 병합합니다.

---

## 2. 우선순위 (Priority)
설정은 아래로 갈수록 생략된 값을 채워주는(Fallback) 역할을 하며, **가장 위쪽에 명시된 값이 최우선순위(Highest Priority)**를 가집니다. 즉 위쪽의 설정이 아래쪽의 설정을 덮어씁니다.

### 전체 우선순위 계층
1. **`.env.local` 내 환경 변수** (최상위, 로컬 개발자 개인 오버라이드용)
2. **`.env.{run_mode}` 내 환경 변수** (예: `.env.dev`, `.env.prod`)
3. **`.env` 내 환경 변수** (공통/기본 환경 변수)
4. **OS/Shell 주입 환경 변수** (CLI에서 `export` 등으로 띄운 변수)
5. **`configs/application-local.toml`**
6. **`configs/application-{run_mode}.toml`**
7. **`configs/application.toml`** (최하위, 글로벌 기본값)

> **💡 주의 (Override 특징):**
> 환경 변수 로딩 시 `dotenvy::from_path_override()`를 사용하므로, 쉘(OS)에서 주입한 환경 변수라 할지라도 `.env.local`이나 `.env` 파일에 동일한 키가 존재하면 **파일의 값으로 덮어써집니다.** 강력한 환경 통제가 가능합니다.

---

## 3. 환경 변수 매핑 규칙 (Environment Variables)
`.env` 파일이나 쉘 환경 변수를 이용해 TOML 설정 계층의 특정 속성 하나만 짚어서 덮어쓸 수 있습니다.

* **Prefix**: 모든 환경 설정용 변수는 `APP__` 로 시작해야만 파서가 인식합니다.
* **Separator**: 계층(Depth)의 구분은 반드시 **더블 언더스코어 (`__`)**를 사용합니다.

### 예시 매핑
| TOML 필드명 (`configs/application.toml`) | 매핑되는 환경 변수명 |
| :--- | :--- |
| `database.url` | `APP__DATABASE__URL` |
| `database.max_connections` | `APP__DATABASE__MAX_CONNECTIONS` |
| `trace.console.filter` | `APP__TRACE__CONSOLE__FILTER` |
| `trace.file.format` | `APP__TRACE__FILE__FORMAT` |

> **왜 더블 언더스코어인가요?**
> `max_connections` 처럼 **필드명 자체에 언더스코어(`_`)가 포함될 경우**, 이를 계층 분리자(Separator)와 착각하지 않게 만들기 위함입니다.

---

## 4. `RUN_MODE` (실행 모드)
* 애플리케이션의 동작 모드는 `RUN_MODE` 환경 변수를 통해 결정되며, 주입되지 않았을 경우 **기본값은 `local`**입니다.
* 예: `RUN_MODE=dev`로 구동 시, 파서는 `application-dev.toml` 파일과 `.env.dev` 환경 변수 파일을 읽어들입니다.

---

## 5. 실행 및 사용 예시
`cargo run` 실행 시 임의 실행은 다음과 같이 처리합니다.

```bash
# 실행 모드를 dev로 강제하고, DB 패스워드와 콘솔 필터만 교체하여 실행
.env.custom
APP__DATABASE__URL="postgres://user:pass@host/db"
APP__TRACE__CONSOLE__FILTER="debug"

RUN_MODE=custom cargo run
```

```bash
# 아니면 local env 를 수정합니다. (최상위 설정)
.env.local
APP__DATABASE__URL="postgres://user:pass@host/db"
APP__TRACE__CONSOLE__FILTER="debug"

cargo run
```
