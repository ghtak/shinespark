# Configuration Guide (`AppConfig`)

Shinespark 애플리케이션은 실행 파일명(또는 크레이트 이름)에 맞춰 동적으로 설정 파일을 인식하는 강력한 계층적 설정 시스템을 지원합니다. 설정은 TOML 파일과 `.env` 환경 변수 파일을 통해 관리되며, 덮어쓰기(Override) 규칙에 의해 최종 값이 결정됩니다.

## 1. 파일 위치 및 로딩
* **설정 파일 (`.toml`)**: 
  - 개발/테스트 중: 워크스페이스의 `configs/` 디렉토리에서 검색
  - 릴리즈 중: 실행 파일이 위치한 경로 기반 검색 (또는 `CONFIG_PATH` 환경 변수로 오버라이드 가능)
* **환경 변수 파일 (`.env` 인픽스/확장자)**: `CONFIG_PATH`에서 동일하게 검색하여 로드합니다.

부트스트랩 시 `AppConfig::load_dotenv()`와 `AppConfig::new()`를 차례로 호출하여 두 가지 소스(File & Env)를 병합합니다.

---

## 2. 네이밍 컨벤션 및 우선순위 (Priority) 
설정 파일들은 모두 **실행 파일의 이름(예: `shinespark-app`)**을 접두사(Prefix)로 가집니다.

설정은 아래로 갈수록 생략된 값을 채워주는(Fallback) 역할을 하며, **가장 위쪽에 명시된 값이 최우선순위(Highest Priority)**를 가집니다. 

### 전체 우선순위 계층 (바이너리가 `shinespark-app`일 경우)
1. **`shinespark-app-local.env` 내 환경 변수** (최상위, 개인 로컬 오버라이드용)
2. **`shinespark-app-{run_mode}.env` 내 환경 변수** (예: `shinespark-app-dev.env`)
3. **`shinespark-app.env` 내 환경 변수** (공통 환경 변수)
4. **OS/Shell 주입 환경 변수** (CLI에서 `export` 등으로 띄운 변수)
5. **`configs/shinespark-app-local.toml`**
6. **`configs/shinespark-app-{run_mode}.toml`** 
7. **`configs/shinespark-app.toml`** (최하위, 글로벌 기본값)

> **💡 주의 (Override 특징):**
> 환경 변수 로딩 시 `dotenvy::from_path_override()`를 사용하므로, 쉘(OS)에서 주입한 환경 변수라 할지라도 `shinespark-app-local.env` 등에 동일한 키가 존재하면 **파일의 값으로 덮어써집니다.**

---

## 3. 환경 변수 매핑 규칙
환경 변수를 이용해 TOML 설정 계층의 특정 속성 하나만 짚어서 덮어쓸 수 있습니다.

* **Prefix**: 모든 환경 설정용 변수는 `APP__` 로 시작해야만 파서가 인식합니다.
* **Separator**: 계층(Depth)의 구분은 반드시 **더블 언더스코어 (`__`)**를 사용합니다.

### 예시 매핑
| TOML 필드명 (`shinespark-app.toml`) | 매핑되는 환경 변수명 |
| :--- | :--- |
| `database.url` | `APP__DATABASE__URL` |
| `database.max_connections` | `APP__DATABASE__MAX_CONNECTIONS` |
| `trace.console.filter` | `APP__TRACE__CONSOLE__FILTER` |
| `trace.file.format` | `APP__TRACE__FILE__FORMAT` |

> **왜 더블 언더스코어인가요?**
> `max_connections` 처럼 **필드명 자체에 언더스코어(`_`)가 포함될 경우**, 이를 계층 분리자(Separator)와 착각하지 않게 만들기 위함입니다. 

---

## 4. `RUN_MODE` (실행 모드)
* 애플리케이션의 동작 모드는 `RUN_MODE` 환경 변수를 통해 결정되며, 주입되지 않았을 경우 **기본값은 `local`**입니다.
* 예: `RUN_MODE=dev`로 구동 시, 파서는 `shinespark-app-dev.toml`과 `shinespark-app-dev.env`를 활성화합니다.

---

## 5. 실행 및 사용 예시
`cargo run` 구동 시 환경 변수를 기반으로 즉석에서 설정을 제어할 수 있습니다.

```bash
# 실행 모드를 custom으로 강제 적용하고 싶을 때
# shinespark-app-custom.env 파일 생성:
APP__DATABASE__URL="postgres://user:pass@host/db"
APP__TRACE__CONSOLE__FILTER="debug"

RUN_MODE=custom cargo run
```

```bash
# 로컬 개발 시에는 -local.env 를 수정합니다 (언제나 최상위 우선순위)
# shinespark-app-local.env 파일 생성:
APP__DATABASE__URL="postgres://user:pass@host/db"
APP__TRACE__CONSOLE__FILTER="debug"

cargo run
```
