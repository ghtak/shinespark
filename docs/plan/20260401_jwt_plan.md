# JWT 구현 계획

## Context

세션 기반 인증(`/identity/session/*`)은 완성. JWT 라우트(`/identity/jwt/*`)는 stub 상태.
`docs/plan/20260418_jwt.md` 스펙 기반으로 구현.

- access_token: HS256, 30분
- refresh_token: 1일, DB 저장 + Rotation

## Phase 5 — DefaultJwtIdentUsecase 구현

### 작업

- [ ] `shinespark-identity/src/infra/default_jwt_ident_usecase.rs` 작성

```rust
pub struct DefaultJwtIdentUsecase<L, U, J, R>
where
    L: LoginUsecase,   // login 인증 위임
    U: UserUsecase,    // refresh 시 user 조회
    J: JwtService,
    R: JwtIdentRepository
```

**login 로직:**
1. `login_usecase.login(command)` → UserAggregate (인증 로직 위임)
2. `jwt_service.create(aggregate)` → JwtTokenPair
3. `sha256(refresh_token)` → token_hash
4. `jwt_repository.save_refresh_token(user_uid, token_hash, expires_at)` 저장
5. JwtTokenPair 반환

**logout 로직:**
1. `jwt_repository.delete_by_user_uid(user_uid)` — claims.sub(UUID) 직접 사용

**refresh 로직:**
1. `jwt_service.verify(token)` → claims (token_type == "refresh" 검증)
2. `jwt_repository.find_refresh_token(sha256(token))` → 존재 + 만료 확인
3. `user_usecase.find_user(FindUserQuery::by_uid(claims.sub))` → UserAggregate
4. 기존 token 삭제 (`delete_by_user_uid(claims.sub)`)
5. 새 JwtTokenPair 생성 및 저장
6. 반환

pub struct AppContainer {
    // 기존 필드 유지
    pub jwt_ident_usecase: Arc<dyn JwtIdentUsecase>,
    pub jwt_service: Arc<dyn JwtService>,  // JwtUser Extractor용

impl AppContainer {
    pub fn new(db: Database, config: &AppConfig) -> Self {
        let jwt_service = Arc::new(HS256JwtService::new(&config.jwt));
        let jwt_repository = Arc::new(SqlxJwtIdentRepository::new());
        let jwt_ident_usecase = Arc::new(DefaultJwtIdentUsecase::new(
            user_repository.clone(), password_service.clone(),
            jwt_service.clone(), jwt_repository,
        ));
        Self { .., jwt_ident_usecase, jwt_service }
    }
}

## 전체 E2E 검증 시나리오

```bash
# 1. 로그인
POST /identity/jwt/login
→ 200, { access_token, refresh_token }

# 2. 잘못된 비밀번호
POST /identity/jwt/login (틀린 비번)
→ 400 INVALID_CREDENTIALS

# 3. 토큰 갱신 (Rotation)
POST /identity/jwt/refresh { refresh_token }
→ 200, 새 토큰 쌍

# 4. 이전 토큰 재사용 불가 확인
POST /identity/jwt/refresh (이전 refresh_token)
→ 401

# 5. 로그아웃
POST /identity/jwt/logout (Authorization: Bearer <access_token>)
→ 200

# 6. 로그아웃 후 refresh 불가
POST /identity/jwt/refresh
→ 401
```
