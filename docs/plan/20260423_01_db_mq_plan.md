# PostgreSQL 기반 MQ 설계안

## 목표

- `SELECT FOR UPDATE SKIP LOCKED` 를 이용한 PostgreSQL 기반 메시지 큐 구현
- `Publisher` / `Consumer` trait 으로 향후 Kafka / RabbitMQ 교체 가능하도록 설계
- `shinespark` (core infra) crate 에 배치 — 도메인 없음, 인프라만

## Phase 2 — Contract

```rust
// mq/mod.rs

pub struct Message<T> {
    pub id: Uuid,
    pub topic: String,
    pub payload: T,
    pub created_at: DateTime<Utc>,
}

#[async_trait]
pub trait Publisher<T: Serialize + Send + Sync> {
    async fn publish(&self, topic: &str, payload: T) -> Result<Uuid>;
}

#[async_trait]
pub trait Consumer<T: DeserializeOwned + Send + Sync> {
    async fn poll(&self, topic: &str) -> Result<Option<Message<T>>>;
    async fn ack(&self, id: Uuid) -> Result<()>;
    async fn nack(&self, id: Uuid) -> Result<()>;
}
```

재시도 횟수, visibility timeout 등은 impl 내부 관심사. trait 확장이 필요해지면 그때 재설계.

## Phase 3 — DB 스키마

```sql
CREATE TABLE shs_mq_messages (
    id           UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
    topic        TEXT        NOT NULL,
    payload      JSONB       NOT NULL,
    status       TEXT        NOT NULL DEFAULT 'pending',  -- pending | processing | done | failed
    attempts     INT         NOT NULL DEFAULT 0,
    max_attempts INT         NOT NULL DEFAULT 3,
    locked_at    TIMESTAMPTZ,
    done_at      TIMESTAMPTZ,
    error        TEXT,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_shs_mq_pending
    ON shs_mq_messages (topic, created_at)
    WHERE status = 'pending';
```

### 핵심 쿼리

**poll**
```sql
WITH next AS (
    SELECT id FROM shs_mq_messages
    WHERE  topic    = $1
      AND  status   = 'pending'
      AND  attempts < max_attempts
    ORDER BY created_at ASC
    LIMIT 1
    FOR UPDATE SKIP LOCKED
)
UPDATE shs_mq_messages
SET    status    = 'processing',
       locked_at = NOW(),
       attempts  = attempts + 1,
       updated_at = NOW()
FROM   next
WHERE  shs_mq_messages.id = next.id
RETURNING *;
```

**reaper — visibility timeout 복구** (백그라운드 태스크)
```sql
UPDATE shs_mq_messages
SET    status     = 'pending',
       locked_at  = NULL,
       updated_at = NOW()
WHERE  status    = 'processing'
  AND  locked_at < NOW() - INTERVAL '5 minutes'
  AND  attempts  < max_attempts;
```
